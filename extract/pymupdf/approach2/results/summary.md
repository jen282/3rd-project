# 방안 2 결과 요약 — pymupdf 페이지 전체 렌더링

## 실행 조건

- 대상 PDF: `국사교과서.pdf` (50페이지)
- 라이브러리: pymupdf 1.27.2.3
- 렌더링 배율: 2x (144 dpi)

## 수치 결과

| 항목 | 값 |
|---|---|
| 총 페이지 수 | 50 |
| 렌더링 완료 | **50페이지** |
| 출력 해상도 | 1191×1684 px (전 페이지 동일) |
| 평균 파일 크기 | ~1.44 MB |
| 최대 파일 크기 | ~2.90 MB |
| 오류 | 0건 |
| 소요 시간 | **8.727초** |

## 방안 1과의 차이

| 비교 항목 | 방안 1 | 방안 2 |
|---|---|---|
| 추출 방식 | 내장 이미지 직접 추출 | 페이지 전체 재렌더링 |
| 속도 | 0.037초 | 8.727초 |
| 파일당 크기 | ~150KB (JPEG) | ~1.4MB (PNG) |
| 벡터 포함 | ❌ | ✅ |
| 텍스트 오버레이 포함 | ❌ | ✅ |

## PDF 구조와의 관계

이 PDF는 이미 래스터 이미지 기반이므로 방안 2의 "벡터 포함" 장점은 이 파일에서는 효과가 없다.  
그러나 텍스트 오버레이가 있는 PDF에서는 방안 2가 방안 1 대비 완전한 시각 정보를 제공한다.

## 장단점 (실측)

**장점**
- 페이지의 모든 시각 요소(벡터 선, 수식, 텍스트 렌더링 포함) 캡처
- 출력 해상도 일관성 보장 (배율 조정 가능)
- Vision 모델 입력으로 바로 활용 가능

**한계 (확인됨)**
- 방안 1 대비 235배 느림 (8.7초 vs 0.037초)
- PNG 파일이 커서 저장 공간 소모 (50페이지 × ~1.4MB ≈ 70MB)
- 어떤 영역이 의미 있는 이미지인지 구분 없음 → 후처리 필요
