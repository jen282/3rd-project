# 방안 1 결과 요약 — pymupdf 래스터 이미지 추출

## 실행 조건

- 대상 PDF: `국사교과서.pdf` (50페이지)
- 라이브러리: pymupdf 1.27.2.3

## 수치 결과

| 항목 | 값 |
|---|---|
| 총 페이지 수 | 50 |
| 추출된 이미지 수 | **100개** |
| 오류 | 0건 |
| 소요 시간 | **0.037초** |
| 확장자 | 모두 JPEG |

## 추출 패턴 분석

각 페이지에서 2개의 이미지가 일정하게 추출됨:

| 이미지 유형 | 크기 | 용도 추정 |
|---|---|---|
| `img00` | 1050×1500 px | 페이지 본문 배경 이미지 (전체 레이아웃) |
| `img01` | 838×188 px | 페이지 헤더/푸터 장식 이미지 |

## PDF 구조 파악

이 PDF는 **텍스트 레이어 없이 래스터 이미지로만 구성된 스캔/인쇄물 기반 PDF**임.
- 각 페이지의 본문 자체가 1050×1500 JPEG 이미지 1장
- 별도 벡터 다이어그램이나 텍스트 레이어 없음

## 장단점 (실측)

**장점**
- 속도 압도적으로 빠름 (0.037초 / 50페이지)
- 원본 JPEG 손실 없이 그대로 추출
- 코드 단순

**한계 (확인됨)**
- 이 PDF에서는 모든 이미지가 페이지 전체를 덮는 배경 이미지 → 어떤 부분이 다이어그램인지 구분 불가
- 벡터 요소는 추출 안 됨 (이 PDF에는 없음)
- bbox 정보 없음 → GraphRAG 위치 메타데이터 연결 불가
