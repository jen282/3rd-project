# 방안 3 결과 요약 — pymupdf 블록 위치 기반 크롭 + 메타데이터

## 실행 조건

- 대상 PDF: `국사교과서.pdf` (50페이지)
- 라이브러리: pymupdf 1.27.2.3 + Pillow 12.2.0
- 렌더링 배율: 2x

## 수치 결과

| 항목 | 값 |
|---|---|
| 총 페이지 수 | 50 |
| 크롭된 이미지 블록 | **100개** |
| 주변 텍스트 확보 | **0개 / 100개** |
| 오류 | 0건 |
| 소요 시간 | **7.437초** |

## 블록 구조 분석

페이지당 정확히 2개의 이미지 블록(type==1) 검출:

| 블록 | PDF bbox | 크롭 픽셀 크기 | 의미 |
|---|---|---|---|
| block0 | (24.7, 30.5, 572.9, 813.1) | 1096×1566 px | 페이지 본문 전체 |
| block1 | (0.2, 0.0, 100.9, 22.4) | 201×44 px | 헤더 로고/장식 |

## 주변 텍스트 0개의 원인

이 PDF는 **텍스트 레이어가 없는 이미지 기반 PDF**이므로 `get_text("dict")`의 텍스트 블록(type==0)이 존재하지 않는다.  
→ `surrounding_text`가 모두 빈 문자열

텍스트 레이어가 있는 PDF(디지털 출력 PDF, OCR 적용 PDF 등)에서는 주변 텍스트가 정상 추출됨.

## GraphRAG 노드 구조 예시

```json
{
  "page": 0,
  "block_index": 0,
  "bbox_pdf": [24.72, 30.48, 572.88, 813.12],
  "bbox_px": [49, 60, 1145, 1626],
  "width_px": 1096,
  "height_px": 1566,
  "surrounding_text": "",
  "saved_as": "results/page000_block00.png",
  "concept": "(GPT-4o Vision으로 채울 필드)",
  "relations": []
}
```

## 방안 3의 핵심 가치 (텍스트 레이어 있는 PDF 기준)

- bbox_pdf → 페이지 내 정확한 위치 정보
- surrounding_text → 이미지 주변 텍스트 컨텍스트
- 두 정보를 조합하면 방안 4(GPT-4o Vision)에 넘길 때 정확한 프롬프트 구성 가능

## 장단점 (실측)

**장점**
- 이미지 위치(bbox) 메타데이터 자동 획득
- 방안 4와 직접 연결되는 구조 (GraphRAG 노드 스캐폴딩)
- 텍스트 레이어 있는 PDF에서는 주변 텍스트까지 함께 확보

**한계 (확인됨)**
- 이 PDF처럼 텍스트 레이어 없는 스캔 PDF에서는 `surrounding_text` 효과 없음
- block0가 페이지 전체를 덮는 이미지 → 의미 있는 세부 영역 크롭 미흡
- 벡터 다이어그램은 type==1로 잡히지 않음 (이 PDF에는 해당 없음)
